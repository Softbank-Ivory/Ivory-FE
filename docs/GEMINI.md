## Frontend 개요

- React+TypeScript+Vite 표준 구조를 사용할 것.
- 유지보수성이 좋고, 성능이 좋은 컴포넌트 구조를 사용할 것.
- 반드시 전체적인 계획을 먼저 수립한 뒤에 단계별로 상황을 기록하며 구현할 것.
- 단계 진행 시 이전 단계의 테스트가 정상적으로 실행되는지 확인할 것.
- 사용자의 요청에 의문이 드는 경우가 있거나 구현이 어려운 경우가 있으면 임의로 구현하기 전에 반드시 사용자에게 상세하게 설명하고 확인할 것.

## UI 개요

- 전체적으로 동일한 스타일을 사용하며, 모던하고 친근한 느낌을 유지할 것.
